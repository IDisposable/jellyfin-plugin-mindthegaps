Settings

Choose what the gap scanner looks for. Use the X at the top to close this panel and return to the report.

Complete what you own

Sources that fill in or extend something already in your library: collections and franchises, studios, keywords, record labels, series episodes, an artist's discography, an author's other works, and an owned actor or director's filmography.
Search TheMovieDb for a studio and pick it; the movies it made that you do not own become gaps. For example A24 or Studio Ghibli.
Search TheMovieDb for a keyword (a theme or motif) and pick it; movies tagged with it that you do not own become gaps.
Search Discogs for a record label and pick it; the releases on it that you do not own become gaps. Needs the Discogs token, set under Data sources.

Discover new titles

Sources that surface titles not tied to a specific owned set: recommendations from similar titles, curated TMDB lists, and MDBList community lists.
Comma-separated TMDB list ids; the movies on each that you do not own become gaps. A list id is the number in its themoviedb.org/list/<id> URL. TMDB has no list search, so paste the id.
Search MDBList for a public list (such as a toplist) and pick it; titles on it that you do not own become discovery gaps. Needs the MDBList API key, set under Data sources.

Where to watch

Powers the "Where to watch" links and the report's "Hide items with no sources" filter. Look-ups run on demand per item, or as a background pass from the report (the "Look up where to watch" button), never during the scan. The "Refresh where to watch" scheduled task keeps it current.
How long a looked-up "where to watch" result stays fresh before it is refreshed. A stale result is still shown instantly while a refresh runs behind the scenes, so this only trades how current the data is against how often TMDB is hit. Default 24.

Data sources

Leave blank to use the built-in default key. Get your own v3 key from themoviedb.org/settings/api to use your own request budget.
Create a free API app at trakt.tv/oauth/applications and paste its Client ID. Required for the Trakt cross-check (opt-in per Trakt's terms).
Keyless. Catches episodes a series' configured metadata provider doesn't list.
Your v4 API key from thetvdb.com/dashboard/account/apikey. Required for the TheTVDB cross-check.
A Discogs personal access token (Discogs requires authentication to browse the catalogue). Create one on discogs.com under Settings, Developers. Pick which labels to complete under "What to scan".
A free MDBList API key (mdblist.com, under Preferences). Needed to search and read MDBList lists. Pick which lists to surface under "What to scan".
Posted to when a scan or the "where to watch" pass finishes (a summary, Discord-compatible). Leave blank to disable.

Limits

Caps how many similar titles each owned item contributes (recommended max 100).
A recommended ("similar") title must have at least this many TMDB votes to surface, trimming the obscure long tail of the discovery feed. 0 shows everything; raise it (for example 500 or 1000) to keep only well-known suggestions.
Caps how many missing episodes are listed per show. 0 lists them all.
Filmography and recommendation scans rotate stalest-first within this cap, so coverage builds up over runs; raise this to cover a large cast and crew faster (each person is one cached TMDB lookup; recommended max 10000).
Keeps Creator works actionable on a large library: a cast credit must have at least this many TMDB votes to surface, which drops obscure and unreleased films. 0 shows everything; raise it (for example 500 or 1000) to trim to only well-known films. Directing/writing credits are always shown.
Drops minor (deeply billed) acting roles, so a bit part is not counted as the person's work. 0 keeps any billing; for example 10 keeps only roles billed in the top 10. Does not affect directing/writing.
Forgets which items were scanned recently, so the next scan starts a fresh coverage cycle treating everything as never-scanned. Does not delete any gaps or dismissals. Stale entries for items that left the library are pruned automatically, so you rarely need this.

Virtual items

The plugin can write pathless "virtual" placeholder items into your library so missing entries render greyed-out, the way missing episodes do. It is a temporary stand-in for proper server support; the server will not reconcile or garbage-collect these for you. Mint individual gaps, or a multi-select batch, from the Mint buttons on the report. Use Remove minted movies here to undo everything at once.
